В рамках тестового задания схема PUSH-уведомлений приведена в упрощённом виде, без использования брокеров сообщений и сложной логики повторных отправок.
Основная цель — показать понимание базового процесса: регистрация device token, хранение токенов, формирование и отправка push-уведомлений через внешний провайдер.
В случае роста проекта архитектура может быть расширена за счёт добавления брокера сообщений, асинхронной обработки и дополнительной бизнес-логики
